Qui suis-je ?

Étudiant en informatique

Passé

Originaire de Grenoble, dans l'Ièsre (38), bien qu'étant guidé par les montagnes et la nature, j'ai toujours été passionné par l'informatique. Passion qui s'est renforcée en première lorsque j'ai pour la première fois touché à la programmation dans la matière NSI. J'ai donc naturellement choisi de m'orienter vers une filière informatique après le bac pour en apprendre d'avantage sur le domaine et me perfectionner. J'ai donc choisi le BUT Informatique de l'IUT Lyon 1 de Bourg-en-Bresse dans lequel je me suis spécialisé dans le développement web.

Présent

J'ai 20 ans et suis actuellement en troisième année de BUT informatique parcours réalisation d'applications : conception, développement, validation en spécialisation développement web, je finis ma formation en alternance à l'entrepôt Carrefour Supply Chain de Mâcon en tant que développeur web full stack. Je suis épanoui dans ce que je fais autant sur le plan professionnel qu'académique. Mon parcours m'a permis d'acquérir des compétences solides en développement web, ainsi qu'une expérience pratique enrichissante en entreprise.

Futur

Je suis en attente de réponses pour diverses candidatures en Master mention informatique en formation classique ou alternance. Je souhaite continuer à me perfectionner dans le développement web et acquérir de nouvelles compétences plus approfondies dans le fonctionnement bas niveau du web. Dans le cas où la formation en master ne me plaît pas, je m'expatrierai à l'étranger (probablement nouvelles-calédonie) pour travailler en tant que développeur web full stack pour une durée indéterminée.

Mes projets marquants

Site pour un boulanger / meunier


Contexte :
Dans le cadre de ma SAE de troisième année, je devais en groupe développer ou améliorer un projet existant.
Dans mon cas, le dit projet visait à fournir une solution informatique à un entrepreneur boulanger/meunier opérant dans un moulin traditionnel.
Le boulanger souhaitait pouvoir gérer les recettes de pain, ainsi que la facturation.
Il voulait pouvoir consolider les commandes pour une journée donnée afin de générer les recettes de fabrication des pâtes avec les quantités d'ingrédients appropriées tout en gérant les différents produits et recettes de sa boulangerie.

Mes missions :
J'ai été en charge de la partie back-end du projet, c'est-à-dire la modélisation et la création des différentes entités du projet
J'ai également développé la partie de gestion des recettes, des produits et des commandes par les utilisateurs

Informations techniques :
Ce projet a été réalisé en groupe de 4 personnes avec en PHP (Framework Symfony) et en JavaScript (HTMX pour les requêtes AJAX) avec une base de données MySQL.
Le déploiement du projet se faisait via Git et le suivi de projet se faisait via Trello.

Site de vente de bière


Contexte :
Dans le cadre du module de Symfony / Doctrine de BUT 3 je devais choisir de développer un projet avec un certain nombre de fonctionnalités afin de montrer que j'avais compris le fonctionnement global de symfony

Mes missions :
J'ai été en charge autant sur la partie back-end que front-end dans le projet. J'ai modélisé et créé les différentes entités du projet pour ensuite créer la base de données
J'ai aujouté à l'aide d'une API une quarentaine de bières pour les afficher, lister leur catégories et les afficher sur une carte
J'ai également ajouté un système de panier pour pouvoir commander les bières et les afficher dans un tableau de commande. Le panier était entièrement réactif, on pouvait modifier la quantité de chaque produit et le prix de la commande était modifié en conséquence
J'ai aussi mis en place un système de coffret de bières qui contenait un ensemble de bière de catégories similaires. J'ai enfin développé un système de filtre pour les bières en fonction de leur catégorie, leur prix, leur amertume, couleur...

Informations techniques :
Ce projet a été en groupe de 2 personnes avec en PHP (Framework Symfony et Bootstrap 5 pour le front) et en JavaScript (HTMX pour les requêtes AJAX) avec une base de données MySQL.
Le déploiement du projet se faisait via Git et le suivi de projet se faisait via Trello.

Site de gestion de BUT

Contexte :
Dans le cadre de ma SAE de troisième année, j'ai dû, en groupe, développer un nouveau gros projet et en faire une démonstration en fin d'année. Dans mon cas, le dit projet visait à développer un site web permettant aux étudiants et aux enseignants de visualiser et gérer les informations pédagogiques liées au BUT. Le site devait principalement permettre la visualisation, navigation et l'affectation des compétences acquises tout au long du BUT


Mes missions :
J'ai été en charge de modéliser la base de données et les différentes entités qui allait la contenir et d'effectuer les différentes migrations de la base de données pour le projet.
J'ai également modéliser les différents droits et accès en fonction des types d'utilisateurs du projet (enseignants, chefs de départements, administrateurs...). J'ai développé diverses routes dans l'api permettant de récupérer les informations nécessaires pour les différentes pages du site (récupérer les étudiants d'une promotion ainsi que leur groupe et les moyennes de chacuns d'eux...).


Informations techniques :
Ce projet a été réalisé en groupe de 3 en Python avec le framework Django. La base de données était une base MySQL.
Le site a été déployé sur une VM de L'IUT. Le suivi déploiement du projet se faisait via Git et le suivi de projet se faisait via Trello.
